Reprovision Player

If the player has already been provisioned, there should be a provisioning record in your BSN.cloud network with an affiliated Device Setup package. In various scenarios, there may be a reason to change how your player was set up, or some element of the setup package itself could be causing an issue. For instance, you may want to change the device type to a partner application or add a WiFi module with SSID to a player. For changes like these, you will need to create a new setup package with the appropriate settings and reprovision the device. Creating a new setup package and applying it to the player is also an extremely effective troubleshooting technique that can often resolve an issue.

Note: If you are using a custom network configuration other than DHCP (like a static IP, proxy, etc.), be sure to choose “Use Current Device Settings” when creating the new Device Setup.

After you have created a new setup package with your changed settings, or updated the previous settings, save it to the Device Setup Library. Once you’ve done that, assign the newly created setup package to the player under Admin > Device Provisioning > Select Device > Apply Setup.

If the player is connected to BSN.cloud, Control Cloud, select the player from the network tab, press the cog icon, navigate to control, and then press “Update” in the Reprovision Device section.

If the player isn't connected to BSN.cloud, there are other offline ways to reprovision the device, but we suggest connecting the players to a BSN.cloud network.

Once the reprovisioning is complete, check to see if this step resolved the issue.
